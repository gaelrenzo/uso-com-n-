# Karpathy Guidelines

4 behavioral principles from Andrej Karpathy for better LLM coding. Reduces overcomplication and unnecessary diffs.

## When to Use

- Any coding task with an AI agent
- User wants cleaner, more focused code changes
- Reducing overcomplication in AI-generated code

## The 4 Principles

### 1. Think Before Coding
Understand the problem fully before writing any code. Ask clarifying questions. Review existing code first.

### 2. Simplicity First
Choose the simplest solution that works. Avoid premature optimization. Prefer standard library over custom code.

### 3. Surgical Changes
Make minimal, focused changes. Don't refactor unrelated code. Keep diffs small and reviewable.

### 4. Goal-Driven Execution
Understand the end goal before starting. Break work into clear steps. Verify each step before moving on.

## Quick Reference

```
Before coding:
- Have I understood the problem?
- What's the simplest approach?
- Am I making only necessary changes?
- Does this serve the stated goal?
```
